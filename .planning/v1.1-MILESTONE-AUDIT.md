## Cross-Phase Integration

### Broken Wiring

**INT-03 (HIGH): j/k/Enter row navigation — Affects PARITY-07**

`keys.js:165` `moveRow()` queries `[data-row]`, but navigable rows in `messages.templ`, `aggregate.templ`, `search.templ`, `dashboard.templ` use `data-href`. Only `deletions.templ` uses `data-row`. Result: j/k is a no-op on all main content pages.

**Fix:** `querySelectorAll('[data-row]')` → `querySelectorAll('[data-href]')` in `moveRow()`.

**INT-04 (MEDIUM): s/r sort cycling — Affects PARITY-07**

`keys.js:220,222,236` queries `.sort-header[data-sort-field]` and `.sort-header.active`, but `SortHeader` in `components.templ:58` generates `sortable-header`. Result: `s` and `r` are no-ops.

**Fix:** Align selector — update `keys.js` to query `sortable-header` or update `components.templ` to emit `sort-header`.

**INT-05 (LOW): go.mod bluemonday misclassification**

`bluemonday` marked `// indirect` but directly imported by `sanitize_email.go`. Fix: `go mod tidy`.
