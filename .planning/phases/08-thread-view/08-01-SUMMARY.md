## Decisions Made
- Handler pre-computes all thread metadata (participants, date range, message count) before passing to template, keeping template logic clean
- `hx-target="closest .email-render-wrapper"` on "Load images" link in expanded messages to avoid ID collision when multiple messages are expanded simultaneously in thread context
- Per-message wrapper IDs (`id="email-body-wrapper-{msgID}"`) in thread context vs. the single `id="email-body-wrapper"` used in the standalone message detail view
