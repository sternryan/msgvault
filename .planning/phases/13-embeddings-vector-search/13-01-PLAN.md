    3. Create `internal/store/vectors.go` with these Store methods:

    **InitVectorTable()** — Creates the vec0 virtual table:
    ```sql
    CREATE VIRTUAL TABLE IF NOT EXISTS vec_messages USING vec0(
        message_id INTEGER PRIMARY KEY,
        embedding FLOAT[1536]
    );
    ```
    Call this from InitSchema() so the vector table is created alongside other schema.

    **InsertEmbeddings(embeddings []VectorEntry) error** — Batch-insert vectors. VectorEntry is:
    ```go
    type VectorEntry struct {
        MessageID int64
        Embedding []float32 // 1536 dimensions
    }
    ```
    Use INSERT OR REPLACE to make it idempotent. sqlite-vec accepts []float32 as a blob parameter for the embedding column. Serialize the float32 slice to a []byte (little-endian, 4 bytes per float32) before inserting — sqlite-vec expects raw float32 bytes, not JSON.

    **SearchSemantic(queryEmbedding []float32, limit int) ([]SemanticResult, error)** — Query vec_messages using KNN:
    ```sql
    SELECT message_id, distance
    FROM vec_messages
    WHERE embedding MATCH ?
    ORDER BY distance
    LIMIT ?
    ```
    The `?` for embedding MATCH is the serialized query vector (same float32->bytes format). sqlite-vec returns cosine distance (0=identical, 2=opposite). Convert to similarity: `similarity = 1.0 - (distance / 2.0)`.

    SemanticResult struct:
    ```go
    type SemanticResult struct {
        MessageID  int64
        Similarity float64 // 0.0 to 1.0
    }
    ```

    **HasEmbedding(messageID int64) (bool, error)** — Check if a message is already embedded (for skip logic):
    ```sql
    SELECT 1 FROM vec_messages WHERE message_id = ?
    ```

    **EmbeddingCount() (int64, error)** — Count total embeddings:
    ```sql
    SELECT COUNT(*) FROM vec_messages
    ```
